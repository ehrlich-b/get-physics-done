  <name>Task 1: Formalize Peirce feedback extension and derive corrected sequential product</name>
  <files>derivations/04-peirce-feedback-extension.md</files>
  <action>
    Derive the corrected sequential product that uses self-modeling feedback to restore the Peirce 1-space content. This is Resolution #1 from the user's direction.

    **Setup -- the problem:**
    The naive extension a & b = sum_i lambda_i C_{p_i}(b) fails S3 because sum_i C_{p_i} is the Peirce pinching map, not the identity. The pinching map annihilates V_1(p_1, ..., p_n) (the off-diagonal/Peirce 1-space).

    **The key insight to formalize:**
    The self-modeling cycle is: test a on B, which triggers compression C_{phi(a)} on M, and the feedback from M back to B restores information. Without feedback (bare compression sum), you lose V_1. WITH feedback, the round-trip B -> M -> B provides the V_1 content.

    **Derivation steps:**

    1. **Peirce decomposition review.** For a resolution of unity {p_1, ..., p_n} in a finite-dimensional spectral OUS V, write the Peirce decomposition:
       V = direct_sum_{i <= j} V_{ij}
       where V_{ii} = face(p_i) (the Peirce 2-space for p_i) and V_{ij} for i < j is the Peirce 1-space connecting p_i and p_j.
       The pinching map is: pinch(b) = sum_i C_{p_i}(b) = sum_i P_{ii}(b), which projects onto the diagonal blocks.
       The Peirce 1-space component is: b_1 = b - pinch(b) = sum_{i<j} P_{ij}(b).

    2. **Formalize the feedback map.** The tracking map phi: Proj(B) -> Proj(M) induces a feedback channel. When a = sum lambda_i p_i, testing a on B:
       (a) Triggers compressions C_{phi(p_i)} on M with weights lambda_i
       (b) M's state is updated, producing a response back to B
       (c) The feedback from M provides information about the CORRELATIONS between the p_i sectors -- this is precisely the V_1 content

       Define the feedback map F_phi: V_B -> V_B as the map that takes the Peirce 1-space information from the B-M correlation and returns it to B. The corrected extension is:

       a & b = sum_i lambda_i C_{p_i}(b) + F_phi(a, b)

       where F_phi(a, b) lives in the Peirce 1-space of the spectral resolution of a.

    3. **Constrain F_phi from S3.** For S3 (1 & a = a), with 1 having spectral decomposition 1 = 1*1 (trivially), the corrected product must give 1 & a = C_1(a) + F_phi(1, a) = a + F_phi(1, a). Since C_1 = id, we need F_phi(1, a) = 0. The identity has trivial spectral resolution (single projector p = 1), so its Peirce decomposition has no 1-space. This is automatically satisfied.

       For a general resolution of unity {p_1, ..., p_n} with all lambda_i = 1 (so a = sum p_i = 1): the corrected product gives 1 & b = sum C_{p_i}(b) + F_phi(1, b). We need this to equal b. Since sum C_{p_i}(b) = pinch(b), we need F_phi(1, b) = b - pinch(b) = b_1 (the Peirce 1-space component). BUT a = 1 has spectral decomposition 1 = 1*1 (single projector), not sum p_i. The spectral decomposition of 1 uses p = 1 with lambda = 1, giving C_1(b) = b directly. There is no inconsistency.

       The real S3 test: for a general effect a with nontrivial spectral decomposition a = sum lambda_i p_i (distinct lambda_i), we need (sum lambda_i C_{p_i}(b)) + F_phi(a, b) to be well-defined, bilinear, and satisfy the remaining axioms. The S3 constraint is automatically satisfied because the spectral decomposition of 1 is trivial.

    4. **Determine F_phi from the self-modeling structure.** The self-modeling constraint says: the feedback from M to B carries information about correlations between the spectral projectors of a. In the Peirce decomposition, the V_{ij} component (i != j) encodes correlations between the p_i and p_j sectors. The tracking map phi, which maps Proj(B) -> Proj(M), induces a correlation structure on M that can be "read back" to B.

       Formalize this: for each pair (p_i, p_j), the tracking map creates a correlation phi(p_i), phi(p_j) in M. The self-model M, by tracking both p_i and p_j, has access to the V_{ij}(M) component. The feedback from M to B maps this back to V_{ij}(B).

       The crucial question: what OUS primitive defines the Peirce 1-space "mixing" map? In a Jordan algebra, the operator P_{ij}(b) = {p_i, b, p_j} (the Jordan triple product) extracts the V_{ij} component. But we cannot use the Jordan triple product (circular -- we're deriving Jordan structure).

       **Approach:** The Peirce 1-space V_{ij} is defined ORDER-THEORETICALLY in Alfsen-Shultz theory as the kernel of (C_{p_i} + C_{p_i^perp} - id) intersected with the kernel of (C_{p_j} + C_{p_j^perp} - id). This is an OUS primitive. The projection P_{ij} onto V_{ij} is the map b -> b - C_{p_i}(b) - C_{p_i^perp}(b) restricted to the subspace where C_{p_j}(b) = 0 and C_{p_j^perp}(b) = 0. However, this extraction is not enough -- we need to know HOW MUCH of the V_{ij} component to include, and this is where phi enters.

       The corrected product formula must be derived from the specific structure of the B -> M -> B feedback loop. Work through this for the simplest non-trivial case: B = M_2(C)^sa (the qubit OUS), with spectral decomposition a = lambda_1 P_0 + lambda_2 P_1 where P_0, P_1 are rank-1 projectors.

    5. **Concrete derivation on M_2(C)^sa.** Work in the 4-dimensional real vector space M_2(C)^sa = span{I, sigma_x, sigma_y, sigma_z}. For a = lambda_1 P_0 + lambda_2 P_1:
       - Peirce 2-spaces: V_{00} = span{P_0}, V_{11} = span{P_1}
       - Peirce 1-space: V_{01} = {b in M_2(C)^sa : P_0 b P_0 = 0, P_1 b P_1 = 0} = off-diagonal Hermitian matrices in the P_0/P_1 basis
       - The naive extension gives: a & b = lambda_1 C_{P_0}(b) + lambda_2 C_{P_1}(b) = lambda_1 P_0 b P_0 + lambda_2 P_1 b P_1 (pinching, loses V_{01})
       - For the corrected product: a & b = lambda_1 P_0 b P_0 + lambda_2 P_1 b P_1 + F(a, b) where F(a, b) must live in V_{01}

       The self-modeling feedback: when B tests a = lambda_1 P_0 + lambda_2 P_1, the model M is updated. If phi(P_0) and phi(P_1) are projectors in M, the feedback through M carries the correlation between the P_0 and P_1 sectors. The Peirce 1-space component of b (the part P_0 b P_1 + P_1 b P_0) is the information that the bare compression sum loses.

       The feedback must reconstruct this from M's response. The key OUS primitive is: for each pair of orthogonal projectors (p_i, p_j), the map b -> b - C_{p_i}(b) - C_{p_j}(b) extracts the V_{ij} component (when p_i + p_j = 1, this is b - pinch(b)).

       The corrected product should include a term that weights the Peirce 1-space component. The simplest bilinear form consistent with S3 and sharp agreement is:

       a & b = sum_i lambda_i C_{p_i}(b) + sum_{i<j} f(lambda_i, lambda_j) P_{ij}(b)

       where f is a symmetric function with f(1,1) = 1 (so 1 & b = pinch(b) + b_1 = b, giving S3 when a = 1 via trivial spectral decomposition) and P_{ij} extracts the Peirce 1-space component.

       **Determine f from first principles:** The S3 constraint (1 & a = a) is automatically satisfied because the spectral decomposition of 1 is trivial (single projector). The nontrivial constraint comes from requiring bilinearity and consistency with sharp effects.

       For sharp p (lambda = 1, single projector): p & b = 1 * C_p(b) = C_p(b). Check (no V_1 term because single projector has no off-diagonal sectors).

       For a = (1/2)I = (1/2)(P_0 + P_1): we need (1/2)I & b. The user's key insight is that this should NOT equal (1/2)(C_{P_0}(b) + C_{P_1}(b)) = (1/2)pinch(b). Instead, the self-modeling feedback should restore the off-diagonal part.

       **Explore what f must be:** The function f(lambda_i, lambda_j) determines the Peirce 1-space weighting. In the Luders product, the corresponding weight is sqrt(lambda_i * lambda_j). The question is whether the self-modeling feedback naturally produces this or a different weight.

       Work through the B -> M -> B feedback loop explicitly for B = M_2(C)^sa to determine what f the self-modeling structure produces. If the feedback map is determined by phi and the OUS compression structure, the form of f is constrained.

       **DO NOT assume f = sqrt(lambda_i * lambda_j).** This would import Hilbert space structure. Instead, derive f from the feedback loop and check whether it coincides with sqrt or differs.

       **If f cannot be determined uniquely from OUS + self-modeling:** Document what constraints f satisfies (from S3, bilinearity, sharp agreement, classical limit, positivity) and what additional structure would be needed to pin it down. This is a valid outcome -- it means the self-modeling constraint constrains but does not uniquely determine the product.

    6. **Bilinearity proof.** Show the corrected product is bilinear:
       - Linearity in b: each C_{p_i}(b) is linear in b; each P_{ij}(b) is linear in b (Peirce projection is linear). Sum of linear maps is linear. Check.
       - Linearity in a: the lambda_i enter linearly in the first sum; f(lambda_i, lambda_j) must be specified to check linearity. If f is bilinear in (lambda_i, lambda_j), the full product is bilinear. Constraint: f must be chosen so the first-argument map is linear.

    7. **S3 proof.** Show 1 & a = a for all effects a. As noted in step 3, this is automatic when the spectral decomposition of 1 is the trivial single projector.

    8. **Classical limit.** On simplices (commutative OUS), V_1 = 0, so the Peirce 1-space terms vanish. The corrected product reduces to sum lambda_i C_{p_i}(b) = pointwise product (as proved in Plan 01). Check.

    9. **Sharp effect agreement.** For sharp p: spectral decomposition has a single projector, so only the compression term contributes. p & b = C_p(b). Matches Plan 01.

    10. **Circularity audit.** Verify the corrected product uses only OUS primitives: compressions C_p, Peirce projections P_{ij} (defined order-theoretically via compressions), spectral decomposition, and the tracking map phi.

    **If Resolution #1 reaches an impasse:** If the self-modeling feedback cannot determine f from OUS primitives alone, document the constraints on f and note that Resolution #2 (quadratic representation U_a) would give f = sqrt(lambda_i * lambda_j) but requires motivating U_a from OUS primitives. Do NOT switch to Resolution #2 without documenting why Resolution #1 is insufficient.

    **Notation:** Use ASSERT_CONVENTION header matching Plan 01. Peirce projections: P_{ij} for V_{ij} component. Feedback map: F_phi. Mixing function: f(lambda_i, lambda_j).
  </action>

       Use at least 3 different triples.

    4. **Classical limit test.** Verify corrected product on n=2 simplex: for diagonal a = diag(a1, a2) and diagonal b = diag(b1, b2), corrected_sp(a, b) = diag(a1*b1, a2*b2).

**Domain:** Clifford algebra representation theory / C*-algebraic operator theory / Jordan algebraic measurement formalism
**Researched:** 2026-04-04
**Confidence:** HIGH (Clifford classification), MEDIUM (sequential product complexification mechanism)

### Scope Boundary

METHODS.md covers analytical and computational PHYSICS methods for v11.0: determining whether the observer's C-linear sequential product a * b = sqrt(a) b sqrt(a) on M_n(C)^sa, applied to real Cl(9,0) generators T_a acting on V_{1/2} = R^16, extends the measurement algebra to Cl(9,C). It does NOT cover software tools (see COMPUTATIONAL.md) or the established v6.0-v10.0 results (Peirce decomposition, impossibility theorems, selection argument) which are INPUTS.

**What is NEW vs v10.0:** v10.0 established that algebraic routes within h_3(O) cannot force complexification (impossibility theorems, Phase 30). v11.0 attacks from the observer's side: the observer is M_n(C)^sa (Paper 5), so the observer's arithmetic is C-linear even when acting on real representations. The question is whether this C-linear arithmetic, specifically the sequential product, generates complex operators when composed from real inputs.

---

## Problem Statement

The Peirce operators T_a (a = 1,...,9) generate Cl^+(9,0) = M_16(R) acting on V_{1/2} = R^16. The observer, whose algebra is M_n(C)^sa, composes these via the sequential product:

    T_a * T_b := sqrt(T_a) T_b sqrt(T_a)

where the square root is the unique positive square root in M_n(C)^sa (Luders product). The core question: does the algebra generated by {T_a * T_b : a,b = 1,...,9} under repeated sequential product and C-linear combinations stay within M_16(R), or does it extend to M_16(C) or M_16(C) + M_16(C)?

## Method Details

### Method 1: Clifford Algebra Classification and Complexification

**What:** Use the Atiyah-Bott-Shapiro classification (Bott periodicity) to determine the exact structure of Cl(9,0) and its complexification Cl(9,C) = Cl(9,0) tensor_R C.

**Mathematical basis:**

From the classification table (Lawson-Michelsohn, Spin Geometry, Table I.4.3; Wikipedia "Classification of Clifford algebras"):

- n = 9, p - q = 9 mod 8 = 1
- Cl(9,0) = M_16(R) + M_16(R) (two simple summands, each 256-dimensional)
- Cl^+(9,0) = M_16(R) (the even subalgebra; this is where Spin(9) lives)
- Volume element omega = gamma_1 ... gamma_9 satisfies omega^2 = +1 (since 9 mod 4 = 1 and signature is positive-definite)
- The two summands are the projections P_+ = (1 + omega)/2 and P_- = (1 - omega)/2
- On V_{1/2} = R^16, the representation is the P_+ projection, so omega = +I_16

Complexification:
- Cl(9,C) = Cl(9,0) tensor_R C = M_16(C) + M_16(C)
- Complex spinor module: S_9^C = C^16 + C^16

**Key fact for v11.0:** The complexification Cl(9,C) has TWICE the real dimension of Cl(9,0). If the sequential product generates operators with nontrivial imaginary parts (relative to the M_16(R) embedding), these imaginary parts live in the "i * M_16(R)" complement, and the generated algebra would be (a subalgebra of) M_16(C).

**Confidence:** HIGH. This is textbook Clifford algebra classification, unchanged since Atiyah-Bott-Shapiro (1964). Verified computationally in Phase 29.

### Method 2: Spectral Analysis and Square Root of T_a Operators

**What:** Compute the spectrum of each Peirce operator T_a to determine whether sqrt(T_a) is well-defined, real, or complex.

**Mathematical basis:** The Peirce operators satisfy {T_a, T_b} = (1/2) delta_{ab} I_16 (convention from v10.0). Therefore T_a^2 = (1/4) I_16, and the eigenvalues of T_a are +/- 1/2.

**Critical observation:** T_a is self-adjoint with eigenvalues +1/2 and -1/2. Since T_a has NEGATIVE eigenvalues, T_a is NOT a positive operator. The "square root" in the sequential product sqrt(a) b sqrt(a) is defined for EFFECTS (operators with 0 <= a <= I), not for arbitrary self-adjoint operators.

This is the FIRST decision point for the analysis:

**(a) If the sequential product is restricted to effects:** We must first shift T_a to make it an effect. The natural choice is a_i = (1/2)I + T_a (eigenvalues 0 and 1, so 0 <= a_i <= I). Then:
    
    sqrt(a_i) = sqrt((1/2)I + T_i)

Since (1/2)I + T_i has eigenvalues 0 and 1, its square root has eigenvalues 0 and 1 (same operator: it's a projection!). So sqrt(a_i) = a_i = (1/2)I + T_i. Then:

    a_i * a_j = ((1/2)I + T_i)((1/2)I + T_j)((1/2)I + T_i)

This is a product of THREE real symmetric matrices, hence REAL and SYMMETRIC. No complexification.

**(b) If the sequential product is extended to all self-adjoint operators:** Need to define sqrt(T_a) for indefinite T_a. In M_n(C)^sa, one can use the functional calculus: if T_a has eigenvalues {+1/2, -1/2}, then we can formally define:

    f(T_a) for f(x) = sqrt(x) 

But sqrt(x) is not defined on R for x < 0 unless we extend to C. So sqrt(-1/2) = i/sqrt(2), and sqrt(T_a) would have complex entries. THIS is the potential complexification mechanism.

**Key formula:** If T_a = (1/2)(P_+ - P_-) where P_+ and P_- are the projections onto the +1/2 and -1/2 eigenspaces, then:

    sqrt(T_a) = (1/sqrt(2))(P_+ + i P_-)   (taking principal branch)

or equivalently sqrt(T_a) = (1/sqrt(2)) P_+ + (i/sqrt(2)) P_- (since sqrt(1/2) = 1/sqrt(2) and sqrt(-1/2) = i/sqrt(2)).

This sqrt(T_a) is COMPLEX: it has an imaginary component i * (1/sqrt(2)) P_-.

**The sequential product then becomes:**
    
    T_a * T_b = sqrt(T_a) T_b sqrt(T_a)
              = (1/sqrt(2))^2 (P_+ + iP_-) T_b (P_+ + iP_-)
              (NOT conj transpose, since sqrt(T_a) is not self-adjoint)

Wait -- the sequential product uses sqrt(a) b sqrt(a), NOT sqrt(a) b sqrt(a)^*. For self-adjoint a, the standard Luders product is a * b = sqrt(a) b sqrt(a) where sqrt denotes the unique positive square root when a >= 0. For indefinite a, the definition must be clarified.

**Resolution:** The standard sequential product (Gudder-Greechie) is defined on the effect algebra [0, I], where all operators are positive. Extension to all self-adjoint operators requires a CHOICE of square root convention. The two natural choices are:

1. **Absolute value route:** Use |T_a| = sqrt(T_a^2) = (1/2)I (always real, always positive). Then sqrt(|T_a|) = (1/sqrt(2))I. This gives a trivial sequential product.

2. **Complex functional calculus route:** Use the principal branch sqrt on the spectrum of T_a, allowing complex values. This gives sqrt(T_a) = (1/sqrt(2))(P_+ + iP_-), which IS complex.

3. **Shift-to-effect route:** As in (a) above, shift T_a -> (1/2)I + T_a to make it a projection, then compute the standard sequential product on effects. This stays real.

**Recommendation:** The v11.0 analysis MUST address which definition of "sequential product" is physically motivated for indefinite operators. The answer determines whether complexification occurs. This is not a mathematical ambiguity -- it is a PHYSICAL modeling choice about how the C*-observer performs measurements.

**Confidence:** HIGH for the spectral analysis. MEDIUM for the physical interpretation of the square root convention.

**References:**
- Gudder & Greechie, "Sequential products on effect algebras," Rep. Math. Phys. 49 (2002) 87-111
- van de Wetering, [arXiv:1803.11139](https://arxiv.org/abs/1803.11139), "Sequential Product Spaces are Jordan Algebras"
- Westerbaan, van de Wetering & Westerbaan, [arXiv:1803.08453](https://arxiv.org/abs/1803.08453), "Three characterisations of the sequential product"

### Method 3: C*-Envelope Analysis

**What:** Determine whether the C-linear closure of the real algebra generated by the Peirce operators equals M_16(C).

**Mathematical basis:** The Peirce operators T_a generate Cl^+(9,0) = M_16(R) as an associative algebra over R. The question: what is the C-linear span of M_16(R) inside M_16(C)?

**Theorem (standard linear algebra):** For any real matrix algebra A subset M_n(R), the C-linear span A_C = A + iA inside M_n(C) satisfies:

    A_C = A tensor_R C

In particular:
- If A = M_16(R), then A_C = M_16(R) tensor_R C = M_16(C).

**Implication:** If we allow the observer to form ARBITRARY C-linear combinations of products of T_a operators, the generated algebra is immediately all of M_16(C). But this is trivial and proves nothing -- it just says "complexifying a real algebra gives a complex algebra."

The nontrivial question is: does the SEQUENTIAL PRODUCT (a specific nonlinear operation) generate complex operators from real inputs, or does it stay within the real subalgebra?

**The C*-envelope approach:** A real C*-algebra A has a complexification A_C which is its C*-envelope. For A = M_16(R) viewed as a real C*-algebra, A_C = M_16(C). The question is whether the sequential product provides a PHYSICAL mechanism that generates elements of A_C \ A, i.e., operators with nontrivial imaginary parts.

This connects to the theory of real operator systems (arXiv:2502.12126, Connes' real structure). The observer's C*-algebra is M_n(C)^sa, which contains the real subalgebra M_16(R)^sa as a Jordan subalgebra. The sequential product on M_n(C)^sa is a * b = sqrt(a) b sqrt(a). When restricted to the real subalgebra, does * stay within the real subalgebra?

**Key result:** If a, b are in M_16(R)^sa (i.e., real symmetric matrices), and a >= 0, then sqrt(a) is also real symmetric (positive square root of a positive real symmetric matrix has real entries). Therefore a * b = sqrt(a) b sqrt(a) is also real symmetric. **The sequential product of positive real operators stays real.**

The only way to leave the real subalgebra is if the observer applies the sequential product to a non-positive operator, which requires the complex functional calculus (Method 2, option (b)).

**Confidence:** HIGH for the algebraic structure. The key insight is that positivity preservation is the gatekeeper: positive real operators have real square roots, so the sequential product of positive operators stays real.

**References:**
- [arXiv:1505.04091](https://arxiv.org/pdf/1505.04091), "Structure and applications of real C*-algebras"
- [arXiv:2502.12126](https://arxiv.org/abs/2502.12126), "Real operator systems"
- [arXiv:2502.04137](https://arxiv.org/html/2502.04137), "Injective envelopes of real C*- and AW*-algebras"

### Method 4: Grade Analysis Under Sequential Product

**What:** Track the Clifford grade decomposition of sequential products to determine whether grade mixing can produce elements outside Cl^+(9,0).

**Mathematical basis:** Cl(9,0) has a Z_2-grading (even/odd) and a finer Z-grading by Clifford degree (grade 0 through 9, with grades k and 9-k identified on V_{1/2} since omega = +I_16). The Peirce operators T_a are grade-1 elements of Cl(9,0).

The even subalgebra Cl^+(9,0) = M_16(R) consists of grades {0, 2, 4, 6, 8}. On V_{1/2}, these are identified with grades {0, 2, 4} (since 6 ~ 3, 8 ~ 1 under the omega = I identification, but this identification is at the level of the representation, not the algebra).

**For the sequential product:**
If a = T_a (grade 1), then a^2 = (1/4)I (grade 0). So a is not positive (eigenvalues +/-1/2).

If instead we use the shifted effect a_i = (1/2)I + T_a (grade 0 + grade 1 = mixed grade), then:
    a_i^2 = (1/4)I + T_a + T_a^2 = (1/4)I + T_a + (1/4)I = (1/2)I + T_a = a_i

So a_i is a projection (idempotent), confirming sqrt(a_i) = a_i.

The sequential product a_i * a_j = a_i a_j a_i involves multiplying three mixed-grade elements. The result:

    a_i a_j a_i = ((1/2)I + T_i)((1/2)I + T_j)((1/2)I + T_i)

Expanding:
    = (1/2)I * (1/2)I * (1/2)I + (1/2)I * (1/2)I * T_i + (1/2)I * T_j * (1/2)I + ...

This is a polynomial in I, T_i, T_j with real coefficients, using only Clifford multiplication. Every term is a real linear combination of Clifford monomials. **The result is in Cl(9,0) (real), not Cl(9,C).**

**Explicit computation:**

    a_i a_j = (1/2)I * (1/2)I + (1/2)I * T_j + T_i * (1/2)I + T_i T_j
            = (1/4)I + (1/2)T_j + (1/2)T_i + T_i T_j

    (a_i a_j) a_i = (1/4)I * a_i + (1/2)T_j * a_i + (1/2)T_i * a_i + (T_i T_j) * a_i
    
    = (1/4)((1/2)I + T_i) + (1/2)(T_j(1/2)I + T_j T_i) + (1/2)((1/2)T_i + T_i^2) + T_i T_j((1/2)I + T_i)
    
    = (1/8)I + (1/4)T_i + (1/4)T_j + (1/2)T_j T_i + (1/4)T_i + (1/2)(1/4)I + (1/2)T_i T_j + T_i T_j T_i

Collecting (using T_i^2 = 1/4 I):

    = (1/8 + 1/8)I + (1/4 + 1/4)T_i + (1/4)T_j + (1/2)(T_j T_i + T_i T_j) + T_i T_j T_i
    = (1/4)I + (1/2)T_i + (1/4)T_j + (1/2){T_i, T_j} + T_i T_j T_i

Using {T_i, T_j} = (1/2) delta_{ij} I:

For i != j:
    = (1/4)I + (1/2)T_i + (1/4)T_j + T_i T_j T_i

And T_i T_j T_i = T_i (T_j T_i) = T_i (-(T_i T_j) + (1/2)delta_{ij}I) = -T_i^2 T_j + (1/2)delta_{ij} T_i

For i != j: T_i T_j T_i = -(1/4)T_j

So: a_i * a_j = (1/4)I + (1/2)T_i + (1/4)T_j - (1/4)T_j = (1/4)I + (1/2)T_i

**This is pure real, grade 0 + grade 1, and equals (1/2) a_i.** The sequential product of distinct shifted effects collapses to a scalar multiple of the first effect.

For i = j: a_i * a_i = a_i (trivially, since a_i is a projection).

**Conclusion from grade analysis:** When the sequential product is defined on effects (shifted Clifford generators), the result stays within the real Clifford algebra. No complexification occurs through this route.

**Confidence:** HIGH. The computation is explicit and exact.

### Method 5: Jordan-Algebraic Measurement Formalism

**What:** Analyze the sequential product using the Jordan algebra framework, connecting to the established result that sequential product spaces are Euclidean Jordan algebras (van de Wetering, arXiv:1803.11139).

**Mathematical basis:** The key theorem (van de Wetering 2018/2019):

**Theorem:** A finite-dimensional order unit space with a continuous sequential product satisfying the Gudder-Greechie axioms is a Euclidean Jordan algebra.

**Implication for v11.0:** The observer's measurement algebra, equipped with the sequential product, must be a Euclidean Jordan algebra. The Euclidean Jordan algebras are classified (Jordan-von Neumann-Wigner 1934):

1. M_n(R)^sa (real symmetric matrices)
2. M_n(C)^sa (complex Hermitian matrices)  
3. M_n(H)^sa (quaternionic Hermitian matrices)
4. V_n (spin factors)
5. h_3(O) (the Albert algebra)

Paper 5 already proved the observer's algebra is M_n(C)^sa (type 2). The question is: when the observer (in M_n(C)^sa) measures observables in V_{1/2} via the Peirce operators (which generate M_16(R)^sa as a Jordan subalgebra), does the measurement algebra expand?

**The Jordan subalgebra perspective:** M_16(R)^sa is a Jordan subalgebra of M_16(C)^sa. The sequential product on M_16(C)^sa, when restricted to M_16(R)^sa, stays within M_16(R)^sa (because positive real symmetric matrices have real symmetric square roots). So the Jordan subalgebra M_16(R)^sa is CLOSED under the sequential product.

**What COULD break this closure:** If the observer forms products like a * b where a is in M_16(C)^sa but NOT in M_16(R)^sa. This happens if and only if the observer has access to observables with complex entries -- i.e., if the observer's representation space is already complexified.

This creates a CIRCULARITY: complexification via sequential product requires the observer to already have complex observables on V_{1/2}, which is what we're trying to derive.

**Breaking the circularity -- the key question for v11.0:** Is there a physical mechanism by which the C*-observer (whose INTERNAL algebra is M_n(C)^sa) generates complex observables on V_{1/2} through its INTERACTION with V_{1/2} via Peirce multiplication?

Candidates:
1. **Composition of non-commuting measurements:** If the observer measures T_a then T_b (non-commuting), the post-measurement state involves T_a * T_b = sqrt(T_a) T_b sqrt(T_a). As shown in Method 4, for effects this stays real.

2. **Complex functional calculus on indefinite operators:** If the observer applies f(T_a) for f requiring complex extension (like sqrt on indefinite operators), this generates complex operators. But the physical justification for this specific functional calculus choice is the v11.0 research question.

3. **Tensor product with observer's internal C structure:** The observer's full Hilbert space is C^n (complex). The interaction V_1 tensor V_{1/2} lives in C^1 tensor R^16 = R^16 or C^1 tensor C^16 = C^16. If the observer TENSORS its complex structure onto V_{1/2}, complexification is automatic -- but this is again the question, not the answer.

**Confidence:** HIGH for the Jordan algebra classification and closure results. LOW for the circularity-breaking mechanism -- this is the open problem that v11.0 must address.

**References:**
- van de Wetering, [arXiv:1803.11139](https://arxiv.org/abs/1803.11139), J. Math. Phys. 60, 062201 (2019)
- Westerbaan, van de Wetering & Westerbaan, [arXiv:1803.08453](https://arxiv.org/abs/1803.08453), J. Math. Phys. 59, 082202 (2018)
- Gudder & Greechie, Rep. Math. Phys. 49 (2002) 87-111

### Method 6: Representation-Theoretic Decomposition of Cl(9,0) tensor C

**What:** Analyze the representation theory of Cl(9,C) = Cl(9,0) tensor_R C to understand the target structure for complexification.

**Mathematical basis:**

Cl(9,0) = M_16(R) + M_16(R), with two irreducible representations on R^16 (the two chiral spinors of the REAL Clifford algebra). On V_{1/2} = R^16, only the P_+ summand acts (omega = +I_16).

Complexification:
    Cl(9,C) = Cl(9,0) tensor_R C = (M_16(R) + M_16(R)) tensor_R C = M_16(C) + M_16(C)

The two summands M_16(C) are the two COMPLEX irreducible representations. On V_{1/2}^C = C^16, only one summand acts. But C^16 is NOT yet a Weyl spinor -- for that we need the Spin(10) structure.

**Connection to Spin(10) and Weyl spinors:**
If V_{1/2} = R^16 is complexified to C^16, and a complex structure J_u is chosen (breaking Spin(9) to its stabilizer), then C^16 decomposes:

    C^16 = S_{10}^+ (as a Spin(10) representation under Cl^+(10,0) = M_16(C))

More precisely: Cl(10,0) = M_32(R) and Cl^+(10,0) = M_16(C). The spinor representation of Spin(10) on C^16 = S_{10}^+ is the Weyl spinor. This is the COMPLEXIFIED representation that Paper 7 needs.

**The representation-theoretic path to complexification:**
1. Start: Cl^+(9,0) = M_16(R) acts on R^16
2. Embed: Cl^+(9,0) subset Cl^+(10,0) via the inclusion Spin(9) subset Spin(10)
3. Target: Cl^+(10,0) = M_16(C) acts on C^16

The embedding Spin(9) subset Spin(10) is standard (add one generator). On representations: S_{10}^+ restricted to Spin(9) gives S_9 tensor_R C (the complexified spinor). The complex structure on S_{10}^+ restricted to Spin(9) is NOT Spin(9)-equivariant (Theorem 1 from Phase 30), which is consistent.

**Method for v11.0:** Use the embedding chain

    Cl^+(9,0) = M_16(R) -> Cl^+(10,0) = M_16(C)

and ask: does the sequential product on V_{1/2} provide the TENTH generator needed to promote Cl^+(9,0) to Cl^+(10,0)?

Specifically: Cl^+(10,0) = M_16(C) is generated by Cl^+(9,0) = M_16(R) together with ONE additional generator gamma_10 satisfying:
- gamma_10^2 = I
- {gamma_10, gamma_a} = 0 for a = 1,...,9 (anticommutation with existing generators)
- gamma_10 is NOT in Cl^+(9,0) = M_16(R)

In the standard construction, gamma_10 = i * J_u (or a conjugate), where J_u is the Krasnov complex structure. The question reduces to: does the observer's sequential product generate J_u or an equivalent structure?

From Phase 30, Theorem 2: J_u is NOT in spin(9) (it has grade-3 components). From Theorem 1: no Spin(9)-equivariant complex structure exists. So generating J_u requires symmetry-breaking input -- which is what the observer's measurement (choosing which T_a to measure) provides.

**Confidence:** HIGH for the representation theory. MEDIUM for the connection to the sequential product mechanism.

### Method 7: Direct Computational Verification (16x16 Matrices)

**What:** Numerically construct the Peirce operators T_a as explicit 16x16 real matrices, compute sequential products, and check for complex components.

**Algorithm:**

```python
# Pseudocode for v11.0 computational verification

# Step 1: Construct T_a from Peirce decomposition (from Phase 29)
# T_a are 16x16 real matrices satisfying {T_a, T_b} = (1/2) delta_{ab} I_16
# These are already available from v10.0 computations

# Step 2: Compute effects a_i = (1/2)I + T_i (eigenvalues 0 and 1)
# Verify: a_i is a projection (a_i^2 = a_i)

# Step 3: Compute sequential product on effects
# a_i * a_j = a_i @ a_j @ a_i  (since sqrt(projection) = projection)
# Check: is result real? (should be YES for effects)

# Step 4: Alternative -- compute sqrt(T_a) via complex functional calculus
# Eigendecompose T_a = U @ diag(1/2, ..., -1/2, ...) @ U^T
# sqrt(T_a) = U @ diag(1/sqrt(2), ..., i/sqrt(2), ...) @ U^T  (COMPLEX)
# Compute T_a * T_b = sqrt(T_a) @ T_b @ sqrt(T_a)
# Check: imaginary part nonzero?

# Step 5: Dimension count
# Generate all products: T_a * T_b, (T_a * T_b) * T_c, ...
# At each level, compute the dimension of the real span
# Track: does dimension exceed 256 = dim(M_16(R))?
# If yes: complexification has occurred

# Step 6: Commutant test
# For the generated algebra A, compute {X in M_16(C) : [X, A] = 0 for all A in A}
# If commutant = C*I (trivially): A = M_16(C) (Burnside's theorem)
# If commutant = R*I: A = M_16(R) (no complexification)
```

**Cost:** Each matrix operation is O(16^3) = O(4096). The full exploration requires at most O(256^2) = O(65536) operations for dimension counting. Total runtime: sub-second on any modern machine.

**What this proves vs what it shows:** Numerical computation can VERIFY that a specific construction generates complex operators (or doesn't). It cannot prove the result in generality -- for that, the analytic methods (1-6) are needed. But numerical verification is an essential first step and provides the ground truth for checking analytic arguments.

**Confidence:** HIGH for computational verification. The matrices are small (16x16), the computations are exact (up to machine precision), and the results are deterministic.

## Critical Decision Tree for v11.0

The analysis branches at one key decision point. The methods above converge on this structure:

```
Q1: Does the observer use the sequential product on EFFECTS or on ARBITRARY self-adjoint operators?

  [EFFECTS: a in [0,I]]
    -> sqrt(a) is real for real a >= 0
    -> Sequential product stays in M_16(R)^sa
    -> NO COMPLEXIFICATION via sequential product alone
    -> Must find another mechanism (tensor product with observer's C, etc.)

  [ARBITRARY SELF-ADJOINT: a in M_n(C)^sa, possibly indefinite]
    -> sqrt(a) requires complex functional calculus for indefinite a
    -> sqrt(T_a) is complex (eigenvalues map to +/-1/2 -> 1/sqrt(2), i/sqrt(2))
    -> Sequential product T_a * T_b has complex entries
    -> Q2: Does the generated algebra equal M_16(C)?
      
      [YES] -> Complexification achieved
              Verify: dim >= 257, or commutant = C*I, or explicit i*J_u generated
      
      [NO]  -> Partial complexification only
              Characterize: which subalgebra of M_16(C)?
```

**Recommendation:** Phase the v11.0 work as:
1. Computational verification (Method 7): determine ground truth
2. If complex FC route works: prove analytically via Methods 2 + 3 + 6
3. If effects route is required: prove the CLOSURE result (sequential product of effects stays real), then investigate alternative complexification mechanisms
4. In either case: connect to the representation-theoretic target Cl^+(10,0) = M_16(C)
